توثيق واجهة برمجة التطبيقات (API)

هذه الصفحة توضح مسارات الـ API الخاصة بالنظام. استخدم القائمة الجانبية للتنقل بين نقاط النهاية المختلفة. البادئة العامة لجميع المسارات هي /api/v1 أو /apiv1.

http://127.0.0.1:5000

قسم النماذج (Models)

هذا القسم يوضح بالتفصيل هيكل كائنات البيانات الأساسية التي يتم استخدامها في جميع استجابات الـ API.

Apartment Model

هيكل كائن الشقة الذي يتم إرجاعه في معظم نقاط النهاية المتعلقة بالشقق.

هيكل الكائن (Object Structure)

الحقل (Field) النوع (Type) الوصف (Description)
idIntegerالمعرف الرقمي الفريد للشقة.
titleStringعنوان الإعلان عن الشقة.
descriptionStringوصف تفصيلي للشقة.
addressStringالعنوان الفعلي للشقة.
priceFloatسعر الإيجار.
roomsIntegerعدد الغرف.
bathroomsIntegerعدد الحمامات.
kitchensIntegerعدد المطابخ.
total_bedsIntegerإجمالي عدد الأسرّة في الشقة.
available_bedsIntegerعدد الأسرّة المتاحة حالياً.
residence_typeStringنوع السكن (مثال: "شباب", "عائلات").
whatsapp_numberStringرقم الواتساب للتواصل.
owner_idIntegerمعرف مالك الشقة.
neighborhood_idIntegerمعرف الحي الذي تقع فيه الشقة.
neighborhood_nameStringاسم الحي (يتم جلبه من العلاقة).
is_verifiedBooleantrue إذا تم تأكيد الشقة من قبل المشرف، وإلا false.
imagesArray of Image Objectsقائمة بكائنات الصور الخاصة بالشقة. انظر Image Model.
first_imageStringرابط URL لأول صورة في القائمة لتسهيل العرض.

قسم المصادقة (Authentication API)

POST

/api/v1/auth/register

الوصف: إنشاء حساب مستخدم جديد.

المدخلات (What it takes)

محتوى الطلب (Request Body) - Content-Type: application/json
الحقلالنوعمطلوب؟الوصف
fullNameStringنعمالاسم الكامل للمستخدم.
emailStringنعمالبريد الإلكتروني (فريد).
passwordStringنعمكلمة المرور.
birthDateStringنعمتاريخ الميلاد بالصيغة YYYY-MM-DD.
مثال على الطلب:
{
    "fullName": "علي محمد",
    "email": "ali@example.com",
    "password": "password123",
    "birthDate": "2002-05-15"
}

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 201 Created

مثال على الاستجابة:
{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "uuid": "a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6"
}
❌ الاستجابات الخاطئة (Error Responses)
  • 400 Bad Request: عند نقص حقول مطلوبة. 
    {"error": "الحقول المطلوبة غير مكتملة", "missing": ["email"]}
  • 400 Bad Request: عند استخدام بريد إلكتروني مسجل مسبقاً. 
    {"error": "البريد الإلكتروني مستخدم بالفعل"}
POST

/api/v1/auth/login

الوصف: تسجيل دخول مستخدم موجود والحصول على Token.

المدخلات (What it takes)

محتوى الطلب (Request Body) - Content-Type: application/json
الحقلالنوعمطلوب؟الوصف
emailStringنعمالبريد الإلكتروني للمستخدم.
passwordStringنعمكلمة مرور المستخدم.

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 200 OK

{
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
        "uuid": "a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6",
        "full_name": "علي محمد",
        "email": "ali@example.com",
        "role": "student"
    }
}
❌ الاستجابات الخاطئة (Error Responses)
  • 401 Unauthorized: في حالة كان البريد الإلكتروني أو كلمة المرور غير صحيحة.
    {"error": "Invalid email or password"}
GET

/api/v1/auth/profile

الوصف: جلب بيانات الملف الشخصي للمستخدم الذي قام بتسجيل الدخول.

المدخلات (What it takes)

ترويسة الطلب (Header): يجب إرسال access_token كالتالي:
Authorization: Bearer <YOUR_JWT_TOKEN>

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 200 OK

{
    "uuid": "a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6",
    "full_name": "علي محمد",
    "email": "ali@example.com",
    "role": "student"
}
❌ الاستجابات الخاطئة (Error Responses)
  • 401 Unauthorized: عند عدم إرسال Token أو كان غير صالح.
    {"msg": "Missing Authorization Header"}

قسم الشقق (Apartments API)

POST

/api/v1/apartments/create

الوصف: إنشاء شقة جديدة.

المدخلات (What it takes)

ترويسة الطلب (Header): Authorization: Bearer <YOUR_JWT_TOKEN>
محتوى الطلب (Request Body) - Content-Type: multipart/form-data
الحقلالنوعمطلوب؟الوصف
titleStringنعمعنوان الإعلان.
priceFloatنعمالسعر.
imagesFileلاملفات الصور (يمكن رفع أكثر من ملف).

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 201 Created

{ "message": "Apartment created successfully", "apartment_id": 1 }
❌ الاستجابات الخاطئة (Error Responses)
  • 401 Unauthorized: عند عدم إرسال Token أو كان غير صالح.
  • 500 Internal Server Error: عند حدوث خطأ في الخادم.
GET

/api/v1/apartments/all_apartments

الوصف: جلب قائمة بجميع الشقق.

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 200 OK

[
    { "id": 1, "title": "شقة رائعة", "price": 5000, ... },
    { "id": 2, "title": "شقة أخرى", "price": 4500, ... }
]
GET

/api/v1/apartments/<int:id>

الوصف: جلب بيانات شقة واحدة محددة.

المدخلات (What it takes)

معاملات المسار (URL Parameters)
المعاملالنوعالوصف
idIntegerالمعرف الرقمي للشقة.

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 200 OK

{ "id": 1, "title": "شقة رائعة", "price": 5000, "images": [...] }
❌ الاستجابات الخاطئة (Error Responses)
  • 404 Not Found: إذا لم يتم العثور على شقة بالـ ID المحدد.

قسم الصور (Images API)

POST

/apiv1/upload-image/<int:apartment_id>

الوصف: رفع ملف صورة وربطه بشقة معينة.

المدخلات (What it takes)

ترويسة الطلب (Header): Authorization: Bearer <YOUR_JWT_TOKEN>
معاملات المسار (URL Parameters)
المعاملالنوعالوصف
apartment_idIntegerالمعرف الرقمي للشقة لربط الصورة بها.
محتوى الطلب (Request Body) - Content-Type: multipart/form-data
الحقلالنوعمطلوب؟الوصف
imageFileنعمملف الصورة (png, jpg, jpeg, gif).

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 201 Created

{
    "message": "تم رفع الصورة بنجاح",
    "image_url": "http://127.0.0.1:5000/uploads/c5b1a2d3.jpeg"
}
❌ الاستجابات الخاطئة (Error Responses)
  • 400 Bad Request: إذا لم يتم إرسال ملف أو كان نوعه غير مدعوم.
  • 404 Not Found: إذا كانت الشقة غير موجودة.
GET

/apiv1/uploads/<string:filename>

الوصف: عرض ملف الصورة مباشرة عن طريق اسمها.

المخرجات (What it returns)

ترجع هذه النقطة ملف الصورة مباشرة (e.g., Content-Type: image/jpeg) وليس JSON.

قسم الأحياء (Neighborhoods API)

POST

/api/v1/neighborhoods/create

الوصف: إنشاء حي جديد.

المدخلات (What it takes)

ترويسة الطلب (Header): Authorization: Bearer <TOKEN> للمستخدم الذي لديه صلاحية "مشرف".
محتوى الطلب (Request Body) - Content-Type: application/json
{ "name": "الدقي" }

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 201 Created

{
    "message": "تم إنشاء الحي بنجاح ✅",
    "neighborhood": { "id": 5, "name": "الدقي" }
}
❌ الاستجابات الخاطئة (Error Responses)
  • 403 Forbidden: إذا لم يكن المستخدم مشرفاً.
  • 400 Bad Request: إذا لم يتم إرسال اسم الحي.
GET

/api/v1/neighborhoods

الوصف: جلب قائمة بجميع الأحياء المتاحة.

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 200 OK

[
    { "id": 1, "name": "المهندسين" },
    { "id": 2, "name": "الزمالك" }
]

/api/v1/neighborhoods/<int:id>/delete

الوصف: حذف حي معين.

المدخلات (What it takes)

ترويسة الطلب (Header): Authorization: Bearer <TOKEN> للمستخدم الذي لديه صلاحية "مشرف".
معاملات المسار (URL Parameters)
المعاملالنوعالوصف
idIntegerالمعرف الرقمي للحي.

المخرجات (What it returns)

✅ الاستجابة الناجحة (Success Response)

Status Code: 200 OK

{ "message": "تم حذف الحي بنجاح" }
❌ الاستجابات الخاطئة (Error Responses)
  • 403 Forbidden: إذا لم يكن المستخدم مشرفاً.
  • 404 Not Found: إذا كان الحي غير موجود.
